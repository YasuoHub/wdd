# 问当地 - 完整架构设计方案

## 一、项目概述

### 1.1 项目背景
问当地是一款基于地理位置的即时互帮互助微信小程序，解决用户"需即时获取异地实时信息"的痛点。以"地理位置匹配"为核心，连接求助者与帮助者，通过积分激励机制，实现异地实时信息快速获取。

### 1.2 核心价值
- **对求助者**：快速获取异地实时信息，降低信息获取成本
- **对帮助者**：利用碎片化时间赚取积分，实现价值变现
- **平台价值**：搭建高效即时的互帮互助生态

## 二、系统架构

### 2.1 技术选型
| 层级 | 技术方案 | 说明 |
|------|---------|------|
| 前端框架 | 原生微信小程序 | 性能最优 |
| 后端服务 | CloudBase云函数 | 无服务器架构 |
| 数据库 | CloudBase MongoDB | 文档型数据库 |
| 文件存储 | CloudBase云存储 | 内置CDN加速 |
| 实时通信 | CloudBase实时数据推送 | 聊天实时推送 |
| 管理后台 | Vue.js 3 | 现代化框架 |

### 2.2 系统架构图
```
微信小程序客户端
       ↓
腾讯云 CloudBase
  ├── 云函数(SCF) - 34个
  ├── 云数据库(MongoDB) - 10个集合
  └── 云存储/实时推送
```

## 四、积分系统

### 4.1 积分获取
- 新用户注册：100积分
- 完善信息：30积分
- 每日签到：5→10→20梯度
- 邀请好友：双方各50积分
- 完成任务：悬赏全额
- 好评奖励：额外5%

### 4.2 积分消耗
- 发布需求：最低10积分
- 追加悬赏：任意金额

### 4.3 积分流转
```
发布需求 → 冻结积分 → 任务完成 → 解冻并转账给帮助者
                ↓
           超时/取消 → 退还积分
```

## 五、业务流程

### 5.1 需求生命周期
```
发布 → 匹配中 → 执行中 → 完成 → 结算(24h自动)
  ↓       ↓        ↓
取消   超时     客服介入
(退积分) (退积分)  (判定)
```

### 5.2 匹配算法
1. **距离优先**：最近常活动地点优先
2. **类型匹配**：完全匹配加分
3. **评分权重**：历史好评率影响排序

## 六、安全策略

### 6.1 防刷机制
- 同设备每天最多注册1个账号
- 同IP每小时最多发布3个需求
- 每月最多邀请20人
- 每分钟最多发30条消息
- 登录失败5次锁定10分钟

### 6.2 内容审核
- 敏感词过滤：直接拦截
- 图片审核：机器鉴黄鉴暴
- 需求审核：机审+人审
